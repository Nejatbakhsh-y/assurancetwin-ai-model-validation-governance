# AssuranceTwin AI

**Model Validation and AI Governance Framework for High-Stakes Financial Machine Learning**

AssuranceTwin AI is an end-to-end model validation and AI governance project designed to simulate how a financial institution could independently validate, challenge, document, monitor, and govern a machine learning model used in a high-stakes decision environment.

The project combines model development, independent validation, fairness review, calibration testing, stress testing, drift monitoring, model card documentation, governance reporting, and a Streamlit dashboard into one reproducible GitHub repository.

---

## 1. Project Overview

This project builds a governed machine learning workflow for a mortgage-lending style classification problem using public HMDA loan/application records.

The central idea is simple:

> The best predictive model is not automatically the best governed model.

Instead of selecting a model only by AUC or accuracy, the project evaluates each model across a broader validation and governance framework:

- Predictive performance
- Calibration quality
- Fairness and bias metrics
- Robustness under stress scenarios
- Drift stability
- Explainability stability
- Documentation completeness
- Monitoring readiness

The final output is an **AssuranceTwin Score**, which summarizes whether the model is appropriate for governed use.

---

## 2. Why This Project Matters

Many machine learning projects stop after model training and basic performance reporting. In regulated or high-impact environments, that is not sufficient.

A credible model governance process must answer questions such as:

- Is the model conceptually sound?
- Does the model perform well on independent validation data?
- Does it create material group-level fairness concerns?
- Are predicted probabilities calibrated?
- Does model behavior remain stable across time?
- How does the model behave under adverse stress scenarios?
- Are limitations documented clearly?
- Is there a monitoring plan after deployment?
- Would a model risk committee approve, conditionally approve, or reject the model?

This project is designed to demonstrate those capabilities in a practical, portfolio-ready format.

## 8. Calibration and Explainability

### Calibration

Calibration analysis checks whether predicted probabilities are reliable.

Calibration script:

`scripts/08_calibration_analysis.py`

Outputs:

- `reports/tables/calibration_summary.csv`
- `reports/figures/calibration_curve.png`

A model with strong discrimination can still be poorly calibrated. In financial decision systems, probability quality matters because model outputs may influence thresholds, reviews, approvals, pricing, or escalation decisions.

### Explainability

Explainability analysis evaluates the drivers of model decisions and whether explanations remain stable across groups, time periods, and model types.

Explainability script:

`scripts/09_explainability_stability.py`

Methods include:

- Feature importance
- Permutation importance
- SHAP values, where available
- Group-level explanation comparison
- Explanation stability across time splits

Outputs:

- `reports/figures/feature_importance.png`
- `reports/figures/shap_summary.png`
- `reports/tables/explanation_stability.csv`

The governance question is not only whether the model can be explained, but whether those explanations are stable and credible.

## 10. AssuranceTwin Score

The AssuranceTwin Score combines predictive performance and governance readiness into one summary score.

Scoring script:

`scripts/12_assurancetwin_score.py`

Score components:

- Predictive Performance Score
- Calibration Score
- Fairness Score
- Robustness / Stress-Test Score
- Drift Stability Score
- Explainability Stability Score
- Documentation Completeness Score
- Monitoring Readiness Score

Formula:

```text
AssuranceTwin Score =
0.20 * Performance
+ 0.15 * Calibration
+ 0.20 * Fairness
+ 0.15 * Robustness
+ 0.10 * Drift Stability
+ 0.10 * Explainability Stability
+ 0.05 * Documentation
+ 0.05 * Monitoring Readiness
```

Outputs:

- `reports/tables/assurancetwin_scorecard.csv`
- `reports/figures/assurancetwin_score_radar.png`

The AssuranceTwin Score is intended as a model risk committee summary indicator, not as a replacement for detailed validation review.

## 15. Limitations

This project is a governance simulation and should not be interpreted as a production lending system.

Main limitations:

- The target variable is derived from public HMDA action fields and is simplified for modeling.
- HMDA public data may not contain all underwriting variables used by real institutions.
- Protected-attribute handling is included for fairness review, not for production decision automation.
- Synthetic stress scenarios are illustrative and not institution-specific macroeconomic forecasts.
- The dashboard is designed for local demonstration, not deployed production monitoring.
- The AssuranceTwin Score is a structured governance summary, not a regulatory approval mechanism.
- Any real-world use would require legal, compliance, business, model risk, and data governance review.
